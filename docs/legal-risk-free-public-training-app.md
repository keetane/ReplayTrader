# 無料公開トレードトレーニングアプリ 法的リスク検証

作成日: 2026-07-05

## 前提

- 実注文は一切行わない。
- 投資助言、売買推奨、シグナル配信、自動売買は行わない。
- ユーザーが自分で入手した CSV データを使う。
- CSV データはサーバーへアップロードしない。
- GitHub Pages で静的 Web アプリとして無料公開する。
- ユーザー CSV はブラウザ内で読み込んで処理する。
- 開発者は動作確認にテストデータを使うが、一般公開せず、検証前にパスワードを求める形にする。
- アプリは無料公開する。
- 将来的に広告収入を付ける可能性がある。

## 結論

この前提なら、金融商品取引業・投資助言業としてのリスクは低い。さらに、CSV をサーバーへアップロードせず GitHub Pages の静的アプリで処理する方式は、運営者による相場データ取得・保存・再配布を避けられるため、データ面のリスクも比較的低くできる。

ただし、無料公開でも広告が付くと商用性が出るため、データ提供元の利用規約、広告 Cookie、プライバシーポリシー、商標表示のリスクは無視できない。

公開形態は「ブラウザ内で CSV を読み込み、サーバーへアップロードしない」方式が最も安全。サンプルデータを同梱したり、サーバーに CSV を保存したり、他ユーザーと共有できる機能を入れると、データ再配布・個人情報・利用規約違反のリスクが上がる。

## リスク評価

| 項目 | リスク | 評価 |
| --- | --- | --- |
| 実注文なし | 証券業・取引媒介リスク | 低 |
| 投資助言なし | 投資助言・代理業リスク | 低 |
| 無料公開 | 金融規制上の報酬性 | 低から中 |
| 広告収入 | 商用性、広告規約、Cookie | 中 |
| ユーザー CSV | データ入手元の規約違反 | 中から高 |
| CSV 非アップロード | 運営者による再配布・保存 | 低 |
| GitHub Pages 静的公開 | サーバー側データ処理 | 低 |
| テストデータのパスワード保護 | 一般公開・再配布 | 低から中 |
| 実在サンプル相場データ同梱 | データ再配布 | 中から高 |
| TradingView 風 UI | 商標・混同・規約 | 中 |

## 金融規制

チャート表示、CSV リプレイ、仮想売買トレーニングに限定し、個別銘柄の売買推奨やユーザーごとの投資判断を提供しない限り、金融商品取引業や投資助言・代理業に該当する可能性は高くない。

禁止すべき機能・表現:

- 「この銘柄は買い」「ここで売るべき」などの推奨文言
- 勝率ランキングを売買推奨に見せる表示
- ユーザー資産やリスク許容度に応じた個別アドバイス
- 売買シグナル配信
- 自動注文・証券 API 連携
- 他人の資金や口座の運用
- 成果報酬、会員向け推奨銘柄、限定シグナル

安全側の表示:

- 「本サービスは投資判断を提供しません」
- 「売買判断は利用者自身の責任で行ってください」
- 「表示される損益は過去データに基づく仮想計算です」
- 「実際の約定、手数料、スリッページ、税金、信用規制を再現するものではありません」

金融庁は無登録で金融商品取引業を行う者への警告情報を公開しているため、助言性・勧誘性・注文連携を入れる場合は再確認が必要。

## データ利用

最大の未確定リスクは CSV の入手元規約。

ユーザーが自分で入手した CSV を使う場合でも、入手元の規約で次が制限されることがある。

- 保存
- 加工
- アプリへの取り込み
- 商用利用
- 第三者への表示
- 派生データの配布
- スクレイピング

安全な設計:

- CSV はユーザーのブラウザ内で処理する。
- サーバーへアップロードしない。
- GitHub Pages へデプロイする公開ビルドには実在相場 CSV を含めない。
- アプリ側で相場データを配布しない。
- サンプルデータは架空データまたは明確に再配布可能なデータだけにする。
- 利用規約で「利用者はアップロード/読み込みデータについて必要な権利を持つ」と明記する。
- 開発者テストデータは本番公開物から分離する。
- 開発者テスト環境はパスワード保護し、検索エンジンにインデックスさせない。

避けるべき設計:

- 管理者が用意した日本株 1 分足データを配布する。
- ユーザー CSV をサーバーに保存して他端末で共有する。
- ユーザー CSV を他ユーザーへ公開できるようにする。
- TradingView、証券会社、情報ベンダーから取得したデータをサンプルとして同梱する。
- パスワード保護なしでテストデータ入りビルドを公開 URL に置く。

## 開発者テストデータ

開発者が動作確認のために自分で入手した CSV を使うこと自体は、公開配布よりリスクが低い。ただし、そのデータが入った画面やファイルへ第三者がアクセスできると、実質的な再配布や第三者表示と見られる可能性がある。

安全側の運用:

- テストデータ入りの検証環境は本番 GitHub Pages と分ける。
- アクセス前にパスワードを求める。
- テストデータ CSV を GitHub の public repository にコミットしない。
- GitHub Pages の公開ブランチや `dist` にテスト CSV を含めない。
- スクリーンショット、録画、デモ動画に実在データが大きく映る場合も公開前に確認する。
- `robots.txt` や `noindex` は補助策にすぎず、パスワード保護の代替にしない。

パスワード保護は「一般公開していない」ことを示す実務上の防御になる。ただし、データ入手元の規約が第三者表示や商用検証環境での利用を明確に禁じている場合は、パスワード保護だけでは足りない可能性がある。

## 広告収入

広告を付けると、無料でも商用サービスとして見られやすくなる。

追加で必要になる対応:

- プライバシーポリシー
- Cookie/広告識別子の説明
- Google AdSense 等の広告事業者名の開示
- パーソナライズ広告のオプトアウト案内
- EEA/UK/スイス向けには Google 認定 CMP など同意管理
- 未成年向けにならない UI/文言
- 誤クリック誘導の禁止

Google AdSense を使う場合、Google は広告 Cookie、第三者ベンダー、オプトアウト方法をプライバシーポリシーに含めるよう求めている。EEA/UK/スイス向け配信では、Google 認定 CMP と IAB TCF 対応が求められる。

## TradingView 関連

TradingView のデータ、API、スクリーンショット、チャート画像、Pine Script、ブランド表示を使わないこと。

安全な方針:

- チャートライブラリは `lightweight-charts` を使う。
- アプリ名に `TradingView` を含めない。
- 公開説明で「TradingView クローン」「TradingView 互換」と言わない。
- 「リプレイ機能を持つチャートトレーニングアプリ」と説明する。

TradingView の Terms は、TradingView 上の market data の再配布や、明示許可のないサービス/API 商用利用を制限している。一方、Lightweight Charts は公式ページで OSS かつ Apache 2.0 licensed と説明されている。

## 推奨公開方式

### 最小リスク版

- 静的 Web アプリ
- CSV は `<input type="file">` でブラウザ内読み込み
- IndexedDB にローカル保存
- サーバー保存なし
- サンプルは架空データのみ
- 広告なし、または非パーソナライズ広告から開始
- GitHub Pages 公開ビルドに実在 CSV を含めない

### 広告あり版

- 上記に加えて、プライバシーポリシー、Cookie 表示、広告ベンダー開示を追加
- EEA/UK/スイスからのアクセスを許すなら CMP を導入
- 日本向け限定なら、アクセス制限または少なくとも言語・利用規約で対象地域を明示

## 実装前チェックリスト

- [ ] アプリが実注文機能を持たない。
- [ ] 売買推奨、シグナル、自動売買がない。
- [ ] ユーザー CSV をサーバーへ送らない。
- [ ] GitHub Pages 公開ビルドに実在 CSV を含めない。
- [ ] サンプルデータは架空データまたは再配布可能データだけ。
- [ ] 開発者テストデータは public repository にコミットしない。
- [ ] テストデータ入り検証環境はパスワード保護する。
- [ ] アプリ名・説明文に TradingView 商標を使わない。
- [ ] `lightweight-charts` のライセンス表示を同梱する。
- [ ] 利用規約にデータ権利確認、免責、禁止事項を入れる。
- [ ] プライバシーポリシーを用意する。
- [ ] 広告を入れる場合は Cookie/広告識別子/オプトアウトを説明する。
- [ ] Google AdSense を使う場合は AdSense ポリシーと EU 同意要件を確認する。
